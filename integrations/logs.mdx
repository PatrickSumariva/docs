A calcANTT registra cada chamada de API feita com suas chaves de API nos logs de integração, dando a você visibilidade total das requisições externas. Use os logs para monitorar a saúde do uso, diagnosticar falhas e verificar se suas integrações estão se comportando conforme esperado.

## O que é registrado

A página exibe um cabeçalho com **indicadores agregados**, uma **lista de requisições** recentes e um **painel de detalhes** que abre ao clicar em qualquer linha.

### Indicadores de resumo

Quatro cartões mostram a saúde do tráfego no período filtrado:

| Indicador | Descrição |
| --- | --- |
| Total de requisições | Total de chamadas recebidas no período. |
| Sucesso (2xx) | Quantidade de requisições com resposta HTTP `2xx`. |
| Erros (4xx + 5xx) | Quantidade de requisições com resposta de erro (cliente ou servidor). |
| Latência média | Tempo médio de resposta, em milissegundos, no período. |

### Colunas da lista de requisições

Cada linha da lista representa uma requisição:

| Coluna | Descrição |
| --- | --- |
| Status | Código HTTP de resposta (ex.: `200`, `201`, `400`, `404`). |
| Aplicativo | Nome da chave de API que originou a requisição (campo `nome` da chave). |
| Endpoint | Método HTTP e caminho chamado (ex.: `POST /api/public/v1/cargas`). |
| IP | Endereço IP de origem da requisição. |
| Início | Data e hora em que a requisição foi recebida. |
| Duração | Tempo total de resposta, em milissegundos. |
| Empresa | Nome da empresa proprietária da chave de API. |

### Painel de detalhes da requisição

Ao clicar em uma linha, abre um painel com três abas:

<CardGroup cols={3}>
  <Card title="Requisição" icon="arrow-right-to-bracket">
    Mostra **método**, **endpoint**, **IP** e **User-Agent**, além do **body** enviado pelo cliente em JSON, com opção de exibir formatado e botão para copiar.
  </Card>

  <Card title="Resposta" icon="arrow-right-from-bracket">
    Mostra o **status HTTP**, o **tempo** total da resposta e o **body** retornado pela API, também com opções de visualização formatada e cópia.
  </Card>

  <Card title="Linha do tempo" icon="clock">
    Marca o instante em que a **requisição foi recebida** e em que a **resposta foi enviada**, com o **total** decorrido entre os dois eventos.
  </Card>
</CardGroup>

<Warning>
  Quando a requisição falha, o **body enviado pelo cliente não é salvo** (mesmo que tenha sido transmitido), por proteção dos dados. As demais informações (status, IP, headers, tempo e body da resposta) continuam disponíveis no log para diagnóstico.
</Warning>

## Acessando os logs

<Steps>
  <Step title="Abra a página de logs">
    No menu superior, vá em **Integrações → Logs de integração**.
  </Step>
  <Step title="Filtre o período e os critérios">
    Clique em **Filtros** para refinar a lista por status, endpoint, período ou outros critérios. O período ativo aparece em destaque ao lado do botão, junto com o contador de requisições.
  </Step>
  <Step title="Acompanhe atualizações em tempo real">
    O switch **Auto** ativa a atualização automática da lista. O rótulo **Atualizado às HH:MM:SS** indica quando os dados foram carregados pela última vez.
  </Step>
  <Step title="Exporte se precisar">
    Use o botão **Exportar** para baixar os logs filtrados, útil para auditorias ou análise offline.
  </Step>
  <Step title="Abra uma entrada de log">
    Clique em qualquer linha para abrir o painel lateral de detalhes, com as abas **Requisição**, **Resposta** e **Linha do tempo**.
  </Step>
</Steps>

<Note>
  O acesso a **Logs de integração** requer que a funcionalidade `LOGS_INTEGRACAO` esteja habilitada no seu plano. Entre em contato com o suporte se esta página não estiver disponível na sua conta.
</Note>

## Logs de webhooks e integrações do marketplace

Os **Logs de integração** mostram apenas as requisições recebidas via **chave de API**. As demais integrações têm suas próprias trilhas de auditoria, acessadas pelo card de cada uma em **Integrações → API & Marketplace**:

| Integração | Onde ver os logs |
| --- | --- |
| Webhooks | **Integrações → API & Marketplace** → card do webhook → aba **Logs** |
| Lincros | **Integrações → API & Marketplace** → card da Lincros → aba **Logs** |
| Provedores de KM (HERE, Google, TomTom, Mapbox) | **Integrações → API & Marketplace** → card do provedor → aba **Logs** |

Em cada card, a aba **Logs** lista as chamadas e eventos recebidos ou enviados para aquela integração específica, útil para diagnosticar entregas de webhook que falharam ou conferir consultas feitas a um provedor de mapas.

## Trilha de auditoria por carga

Os logs de integração mostram a atividade de cada requisição feita com chaves de API. Para uma visão mais ampla do que aconteceu com um registro de carga específico (incluindo cálculos, edições e cancelamentos), abra **Cargas**, selecione a carga e veja a aba **Logs**.
