A tabela de fretes da ANTT é a regulamentação oficial brasileira que estabelece os **preços mínimos** que as transportadoras podem cobrar pelo transporte rodoviário. Emitida pela ANTT (Agência Nacional de Transportes Terrestres), define um piso regulado para cada combinação de tipo de carga, número de eixos do veículo e distância da rota, tornando legalmente obrigatório que embarcadores e transportadoras verifiquem se qualquer valor de frete declarado atende ou supera o mínimo aplicável.

A calcANTT mantém um banco de dados versionado dessas tabelas para que cada cálculo sempre use as taxas que estavam legalmente em vigor na data do embarque, não apenas a tabela atual.

## Estrutura da tabela

Cada tabela ANTT na plataforma é armazenada como um registro `TabelaAntt` com os seguintes campos de nível superior:

| Campo | Descrição |
| --- | --- |
| `codigo` | Identificador numérico da tabela na plataforma, exibido com o prefixo `#` (ex.: `#1234`). |
| `categoria` | Categoria regulatória da ANTT à qual a tabela pertence: `A`, `B`, `C` ou `D`. Cada letra agrupa uma combinação específica de tipo de operação e veículo definida na Resolução ANTT nº 5.867/2020. |
| `nome` | Nome legível exibido em toda a interface. |
| `vigenciaInicio` | Data de início do período de vigência da tabela (`yyyy-MM-dd`). |
| `vigenciaFim` | Data de fim da vigência (`yyyy-MM-dd`), ou `null` se a tabela ainda estiver em vigor sem expiração definida. |
| `status` | Estado atual do ciclo de vida (veja abaixo). |
| `altoDesempenho` | `true` quando esta tabela aplica o padrão de veículo de alto desempenho (`PadraoTabela`). |
| `composicao` | `true` quando esta tabela aplica o padrão de composição / semirreboque duplo. |

### Valores de status da tabela

## Itens de taxa e valores por eixo

Cada tabela contém uma lista de `itens`: uma entrada por combinação de `tipoCarga` × `componente`. Cada `ItemTabela` especifica:

- **`tipoCargaCodigo` / `tipoCargaNome`**: a classificação de carga à qual esta taxa se aplica.
- **`componenteCodigo` / `componenteNome`**: o componente de taxa (ex.: `CCD` para o componente de custo baseado em distância).
- **`unidadeCalculo`**: como a taxa é aplicada:
  - `POR_KM`: a tarifa é multiplicada pela distância da rota em quilômetros.
  - `FIXO`: um valor fixo aplicado independentemente da distância.
- **`valores`**: um array de entradas `ValorEixo`, uma por número de eixos (`numEixos`), cada uma com o `valor` monetário para aquele tamanho de veículo.

## Componentes do cálculo

Quando você executa um cálculo, o motor resolve cada `ItemTabela` correspondente em um `ComponenteCalculado` no resultado:

| Campo | Descrição |
| --- | --- |
| `codigo` | Código do componente (`CCD`, `CC` ou `RETORNO_VAZIO`). |
| `nome` | Nome do componente exibido nos resultados. |
| `unidade` | `POR_KM`, `FIXO` ou `PERCENTUAL`. |
| `tarifa` | A taxa unitária extraída da tabela ANTT para o `numEixos` selecionado. |
| `valor` | O valor monetário calculado (`tarifa × distância` para `POR_KM`; fixo para `FIXO`). |
| `formula` | Uma string legível mostrando exatamente como o `valor` foi calculado. |

## Generalidades (cobranças percentuais)

Algumas cobranças não são expressas como taxas absolutas, mas como porcentagem de outro componente. Essas aparecem como entradas `Generalidade` na tabela e surgem como componentes `PERCENTUAL` no resultado do cálculo. Cada generalidade referencia:

- `componenteBaseCodigo`: o componente ao qual é aplicada (ex.: `CCD`).
- `percentual`: a taxa aplicável como percentual.

O adicional de retorno em vazio (`RETORNO_VAZIO`) é o exemplo mais comum: quando `retornoVazio` é `true`, o motor adiciona uma porcentagem do componente de distância base ao total e o apresenta como uma entrada `PERCENTUAL` em `componentes` com o `componenteBaseCodigo` ao qual foi aplicado.

## Padrões de tabela (padrões)

Um `PadraoTabela` agrupa tabelas que compartilham a mesma configuração de veículo. Cada padrão tem indicadores booleanos `altoDesempenho` e `composicao`. Os dois principais padrões que você encontrará são:

- **Alto Desempenho (`altoDesempenho: true`)**: tabelas para configurações de veículos de alto desempenho, que têm taxas mínimas diferentes sob a regulamentação.
- **Composição (`composicao: true`)**: tabelas para caminhões em composição (semirreboques duplos e configurações similares).

Quando você habilita os indicadores `altoDesempenho` ou `composicao` na calculadora ou em um registro de carga, o motor automaticamente restringe a seleção de tabelas àquelas cujo `PadraoTabela` corresponde a esses indicadores.

## Como a plataforma seleciona as tabelas

<Info>
  A calculadora e o motor de cálculo de cargas selecionam todas as tabelas ANTT cuja `vigenciaInicio` ≤ `dataCarga` ≤ `vigenciaFim` (ou `vigenciaFim` é `null`) e cujos indicadores `altoDesempenho` / `composicao` correspondem aos seus parâmetros de entrada. Todas as tabelas correspondentes são retornadas no array `tabelas` para que você possa escolher a mais adequada para o seu registro de carga.
</Info>

Os resultados podem incluir mais de uma tabela correspondente, por exemplo, quando uma atualização regulatória introduziu uma nova tabela que se sobrepõe ao final de uma versão anterior. Revise o `tabelaCodigo`, `tabelaNome` e `padraoCodigo` de cada resultado para identificar a versão correta para o seu embarque.

<Tip>
  Sempre verifique a `dataCarga` antes de executar um cálculo. Inserir a data errada faz o motor selecionar uma tabela de um período regulatório diferente, produzindo um valor mínimo que não reflete as taxas legalmente em vigor na data real do embarque.
</Tip>

## Visualizador da tabela de fretes

Dentro da plataforma (após o login), você pode consultar as tabelas ANTT vigentes em **Tabela de fretes**. A página lista as tarifas oficiais por padrão, tipo de carga e número de eixos, e permite **exportar a tabela em formato Excel** para compartilhar com transportadoras, auditores ou clientes, ou para análises offline.
