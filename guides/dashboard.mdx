O painel é sua central operacional na calcANTT. Toda vez que você faz login, ele mostra um panorama da sua operação de frete: quantos registros de carga foram criados, como os cálculos se distribuem entre os canais, se os valores de frete declarados estão atendendo ao mínimo da ANTT e quanto da cota do seu plano foi consumido. Use o seletor **Período** no canto superior direito para limitar a análise aos últimos 7, 30 ou 90 dias.

## Uso e limites do plano

O painel mostra o uso atual dos principais limites do seu plano. Você pode acompanhá-los a qualquer momento pelos indicadores exibidos na plataforma:

<CardGroup cols={3}>
  <Card title="Cota de cálculos" icon="calculator">
    `utilizadoCalculos` de `limiteCalculosMes`: o número de cálculos de frete realizados no ciclo de faturamento atual. Conta cálculos da Calculadora web, dos registros de carga e da API pública.
  </Card>

  <Card title="Cota de requisições" icon="tower-broadcast">
    `utilizadoRequisicoes` de `limiteRequisicoesMes`: total de requisições REST feitas com chaves de API pelas suas integrações externas no ciclo atual.
  </Card>

  <Card title="Chaves de API" icon="key">
    `utilizadoApiKeys` de `limiteApiKeys`: o número de chaves de API ativas que você criou. Chaves adicionais requerem upgrade do plano ou um add-on de Chave de API.
  </Card>

  <Card title="Usuários" icon="users">
    `utilizadoUsuarios` de `limiteUsuarios`: o número de usuários ativos da empresa na plataforma. Convites adicionais ficam bloqueados quando o limite é atingido.
  </Card>

  <Card title="Importação Excel" icon="file-excel">
    `limiteImportacaoCargas`: número máximo de linhas que cada planilha de importação de cargas pode ter. Esse limite é por planilha, não há cota mensal.
  </Card>
</CardGroup>

O contador `diasRestantesCiclo` mostra quantos dias faltam antes de as cotas mensais (`limiteCalculosMes` e `limiteRequisicoesMes`) serem resetadas no início do próximo ciclo de faturamento.

<Info>
  `limiteRequisicoesMes` conta **apenas** as chamadas REST feitas com chaves de API pelas suas integrações externas (ERP, TMS, automações). Cliques na calculadora web e cálculos disparados por registros de carga não consomem `limiteRequisicoesMes`, mas continuam contando para `limiteCalculosMes`.
</Info>

<Tip>
  Quando qualquer limite estiver próximo do teto, acesse **Configurações → Plano** para contratar add-ons ou fazer upgrade. Add-ons podem estender seu `limiteCalculosMes`, `limiteRequisicoesMes`, `limiteApiKeys` ou `limiteImportacaoCargas` sem exigir uma mudança completa de plano. **Configurações → Faturamento** trata apenas das informações de pagamento, faturas e método de cobrança.
</Tip>

## Indicadores do plano trial

Se sua conta estiver em um plano trial, dois indicadores adicionais aparecem com destaque no painel:

- **`trialTerminaEm`**: o último dia do seu período trial (formato `yyyy-MM-dd`).
- **`diasRestantesTrial`**: o número de dias restantes, atualizado diariamente no fuso horário de São Paulo.

Uma barra de progresso reflete quanto do trial você já percorreu, derivada de `duracaoTrialDias`. Quando o trial termina, sua conta é convertida automaticamente para o plano **Free**, que tem limites mensais menores e um conjunto de funcionalidades reduzido em comparação ao trial. Para continuar usando os recursos completos, contrate um plano pago em **Configurações → Plano** antes da expiração.

## Painel de cargas

O cartão **Cargas** resume os registros de frete criados no período selecionado:

- **Total registradas**: total de registros de carga em todos os status (`cargas.total`).
- **Situação**: detalhamento por `situacao`: registros em operação (`CRIADO`) vs. cancelados (`CANCELADO`).
- **Cálculo ANTT**: detalhamento por `statusCalculo`: quantos registros estão `CALCULADO` e quantos permanecem `NAO_CALCULADO` (aguardando cálculo).
- **Frete declarado vs. mínimo ANTT**: detalhamento por `valorAtendido`:
  | Valor | Significado |
  | --- | --- |
  | `ATENDE` | O `valorFrete` declarado atende ou supera o mínimo da ANTT |
  | `NAO_ATENDE` | O valor declarado está abaixo do piso regulatório |
  | `NAO_INFORMADO` | Nenhum `valorFrete` foi declarado; a comparação de conformidade não é possível |
  <Note>
    A comparação de conformidade se aplica apenas a registros de carga que possuem tanto um `valorFrete` declarado **quanto** um cálculo concluído. Registros sem valor de frete declarado sempre aparecem como `NAO_INFORMADO`.
  </Note>
- **Tipos de carga mais frequentes**: um gráfico de rosca mostrando a distribuição dos tipos de carga (`tipoCargaNome`) nos seus registros no período selecionado.

Clique em **Abrir cargas** para navegar diretamente para a página completa de Cargas com seus filtros e grade.

## Painel de cálculos

O cartão **Cálculos de frete** detalha o volume de cálculos por canal que disparou cada cálculo:

| Métrica | Origem |
| --- | --- |
| `calculos.calculadora` | Cálculos disparados pela interface da Calculadora web |
| `calculos.api` | Cálculos feitos pela API pública usando uma chave de API |
| `calculos.cargas` | Cálculos executados em registros de carga (manual ou em lote) |
| `calculos.comAvisos` | Cálculos que retornaram uma ou mais entradas no array `avisos` |

Um gráfico de barras proporcional mostra a participação relativa de cada canal para que você veja rapidamente se a maior parte do uso vem da interface web, dos fluxos de carga ou das integrações via API.

## Painel de integração via API

O cartão **Integração via API** resume as requisições autenticadas feitas com suas chaves de API durante o período selecionado:

- **Requisições**: total de requisições recebidas (`apiIntegracao.totalRequisicoes`).
- **Taxa de sucesso**: porcentagem de requisições que retornaram resposta `2xx` (`taxaSucessoPercentual`).
- **Erros 4xx**: erros do lado do cliente, como parâmetros inválidos, falhas de autenticação ou cota excedida.
- **Erros 5xx**: erros do lado do servidor (`erros5xx`).
