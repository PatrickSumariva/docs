title: "POST /calculadora/calcular | Frete mínimo ANTT"
description: "Envie numEixos, tipoCargaId, distanciaKm e dataCarga para receber as taxas mínimas da ANTT detalhadas por tabela, componente e fórmula."
---

O endpoint `/calculadora/calcular` é o núcleo da API da calcANTT. Ele aceita os parâmetros de rota e veículo e retorna o frete mínimo da ANTT, detalhado por tabela aplicável e componente de custo. Requer a permissão `CALCULAR_FRETE`.

**Endpoint:** `POST /api/public/v1/calculadora/calcular`

## Requisição

```bash
curl -X POST https://your-domain/api/public/v1/calculadora/calcular \
  -H "Authorization: Bearer calt_live_sua_chave_aqui" \
  -H "Content-Type: application/json" \
  -d '{
    "numEixos": 6,
    "distanciaKm": 500,
    "tipoCargaId": 1,
    "altoDesempenho": false,
    "composicao": false,
    "retornoVazio": false,
    "dataCarga": "2026-05-23"
  }'
```

### Parâmetros do corpo da requisição

<ParamField body="numEixos" type="number" required>
  Número de eixos do veículo. Valores válidos: `2`, `3`, `4`, `5`, `6`, `7`, `8`, `9`.
</ParamField>

<ParamField body="distanciaKm" type="number">
  Distância da rota em quilômetros. Use `0` se fornecendo `cepOrigem` e `cepDestino`.
</ParamField>

<ParamField body="cepOrigem" type="string">
  CEP de origem (com ou sem máscara). Usado quando `distanciaKm` é `0`. Requer uma integração de mapas ativa.
</ParamField>

<ParamField body="cepDestino" type="string">
  CEP de destino (com ou sem máscara). Usado quando `distanciaKm` é `0`. Requer uma integração de mapas ativa.
</ParamField>

<ParamField body="tipoCargaId" type="number" required>
  ID do tipo de carga. Busque os IDs válidos em `GET /calculadora/tipos-carga`.
</ParamField>

<ParamField body="altoDesempenho" type="boolean" required>
  `true` para configurações de veículos de alto desempenho (aplica o padrão de tabela ANTT correspondente).
</ParamField>

<ParamField body="composicao" type="boolean" required>
  `true` para configurações de veículos em composição (bitrem, rodotrem).
</ParamField>

<ParamField body="retornoVazio" type="boolean" required>
  `true` para incluir o adicional de retorno em vazio no total.
</ParamField>

<ParamField body="dataCarga" type="string" required>
  Data da carga no formato `yyyy-MM-dd`. Determina qual versão da tabela ANTT estava vigente nessa data.
</ParamField>

### Campos da resposta

<ResponseField name="parametros" type="object">
  Eco dos parâmetros da requisição usados para o cálculo.
</ResponseField>

<ResponseField name="tabelas" type="array">
  Uma ou mais tabelas ANTT que estavam ativas na `dataCarga`. Cada tabela contém:

  <Expandable title="Campos do objeto tabela">
    <ResponseField name="tabelaCodigo" type="string">
      Código único para a versão da tabela ANTT (ex.: `2026-Q1`).
    </ResponseField>

    <ResponseField name="componentes" type="array">
      Componentes de custo que compõem o frete. Cada um tem `codigo`, `nome`, `unidade` (`POR_KM` ou `FIXO`), `tarifa`, `valor` e `formula`.
    </ResponseField>

    <ResponseField name="subtotalIda" type="number">
      Total do frete de ida (soma de todos os componentes).
    </ResponseField>

    <ResponseField name="valorRetornoVazio" type="number">
      Adicional de retorno em vazio. `0` quando `retornoVazio` é `false`.
    </ResponseField>

    <ResponseField name="valorTotal" type="number">
      Total final: `subtotalIda + valorRetornoVazio`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="avisos" type="array">
  Mensagens de aviso (ex.: quando uma combinação de parâmetros é incomum). Pode estar vazio.
</ResponseField>

<ResponseField name="cacheHit" type="boolean">
  `true` quando o resultado foi servido do cache. Resultados em cache são idênticos a cálculos recentes.
</ResponseField>

<ResponseField name="calculadoEm" type="string">
  Timestamp ISO 8601 de quando o cálculo foi realizado.
</ResponseField>

## Observações

<Note>
  Quando múltiplas tabelas ANTT estão ativas na `dataCarga` (ex.: variantes padrão e alto desempenho), a resposta contém múltiplos objetos em `tabelas`. Revise cada um e selecione o mais adequado para a sua carga.
</Note>

<Tip>
  Use `GET /calculadora/tipos-carga` para consultar os valores válidos de `tipoCargaId` antes de chamar este endpoint.
</Tip>
