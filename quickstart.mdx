---
title: "Crie sua conta, gere uma chave de API e faça seu primeiro cálculo"
description: "Crie sua conta, gere uma chave de API com a permissão CALCULAR_FRETE e faça seu primeiro cálculo de frete em conformidade com a ANTT em minutos."
---

Este guia mostra como criar uma conta, gerar uma chave de API com a permissão correta e realizar seu primeiro cálculo de frete na API pública da calcANTT. Ao final, você terá uma requisição `curl` funcional que retorna o frete mínimo totalmente detalhado.

<Steps>
  <Step title="Cadastre-se e ative sua conta">
    Acesse a página de cadastro da calcANTT e registre sua empresa. Após enviar o formulário, verifique seu e-mail pelo link de ativação e clique nele para confirmar sua conta.

    Após ativar, faça login no painel. Sua conta começa no plano trial, que dá acesso à calculadora e à API pública enquanto você avalia a plataforma.
  </Step>
  <Step title="Gere uma chave de API">
    As chaves de API autenticam suas requisições à API pública da calcANTT. Para criar uma:

    1. No menu superior, vá em **Integrações → API & Marketplace**.
    2. Clique em **Nova chave de API**.
    3. Dê um nome descritivo para a chave (por exemplo, `ERP produção`).
    4. Em **Permissões**, marque **`CALCULAR_FRETE`**: isso concede acesso aos endpoints de cálculo de frete.
    5. Clique em **Criar**.

    A chave completa (`chaveCompleta`) é exibida **apenas uma vez**. Copie-a imediatamente e armazene em um gerenciador de segredos ou variável de ambiente.

    <Note>
      Sua chave de API é uma credencial secreta. Nunca a inclua no controle de versão nem a exponha em código que roda no navegador. Se uma chave for comprometida, acesse **Integrações → API & Marketplace** e **rotacione** a chave (ou revogue-a e gere uma nova).
    </Note>

    As chaves seguem o formato `calt_live_<token>`. O prefixo `calt_` identifica uma chave da calcANTT.
  </Step>
  <Step title="Faça seu primeiro cálculo de frete">
    Envie uma requisição `POST` ao endpoint de cálculo. O exemplo abaixo calcula o frete mínimo para um veículo de seis eixos percorrendo 500 km, com tipo de carga ID `1`, na data `2026-05-23`:

    ```bash
    curl -X POST https://app.calcantt.com.br/api/public/v1/calculadora/calcular \
      -H "Authorization: Bearer calt_live_sua_chave_api" \
      -H "Content-Type: application/json" \
      -d '{
        "numEixos": 6,
        "distanciaKm": 500,
        "tipoCargaId": 1,
        "altoDesempenho": false,
        "composicao": false,
        "retornoVazio": false,
        "dataCarga": "2026-05-23"
      }'
    ```

    Substitua `calt_live_sua_chave_api` pela chave de API que você gerou no passo anterior.

    **Referência dos campos da requisição:**

    | Campo | Tipo | Descrição |
    | --- | --- | --- |
    | `numEixos` | inteiro | Número de eixos do veículo |
    | `distanciaKm` | número | Distância da rota em quilômetros. Use `0` para calcular por CEP de origem/destino |
    | `tipoCargaId` | inteiro | ID do tipo de carga, obtenha os IDs válidos em `GET /api/public/v1/calculadora/tipos-carga` |
    | `altoDesempenho` | booleano | `true` para veículos de alto desempenho (PBT/PBTC) |
    | `composicao` | booleano | `true` para conjuntos de veículos em composição |
    | `retornoVazio` | booleano | `true` para incluir o adicional de retorno em vazio |
    | `dataCarga` | string | Data da carga no formato `yyyy-MM-dd`: determina a versão da tabela ANTT aplicável |
    | `cepOrigem` | string | CEP de origem (opcional, usado quando `distanciaKm` é `0`) |
    | `cepDestino` | string | CEP de destino (opcional, usado quando `distanciaKm` é `0`) |
  </Step>

    **Principais campos da resposta:**

    | Campo | Descrição |
    | --- | --- |
    | `sucesso` | `true` quando a requisição foi processada com sucesso |
    | `calculado` | `true` quando ao menos uma tabela vigente foi calculada |
    | `mensagem` | Resumo legível do resultado |
    | `parametros` | Eco dos parâmetros de entrada usados no cálculo |
    | `tabelas` | Array de resultados, uma entrada por tabela ANTT vigente na `dataCarga` |
    | `tabelas[].tabelaCodigo` / `tabelaNome` | Identificação da tabela ANTT aplicada |
    | `tabelas[].padraoCodigo` / `padraoNome` | Padrão tarifário da tabela |
    | `tabelas[].menorValor` | `true` na tabela de menor `valorTotal` entre as retornadas |
    | `tabelas[].valorAtendido` | `true`/`false` se o `valorFrete` informado atende ao piso; `null` quando `valorFrete` não foi enviado |
    | `tabelas[].componentes` | Componentes de custo detalhados com `tarifa`, `unidade`, `valor`, `numEixosUsado` e `formula` |
    | `tabelas[].subtotalIda` | Total da ida (soma dos componentes de ida) |
    | `tabelas[].valorRetornoVazio` | Adicional de retorno em vazio (`0` quando `retornoVazio` é `false`) |
    | `tabelas[].valorTotal` | Frete mínimo final para esta tabela: `subtotalIda + valorRetornoVazio` |
    | `tabelas[].detalhesOperacao` | Memória de cálculo legível: operação, distância, coeficientes e fórmulas |
    | `calculadoEm` | Timestamp ISO 8601 de quando o cálculo foi realizado |
    | `cacheHit` | `true` se o resultado veio do cache, em vez de ter sido recalculado |
    | `avisos` | Avisos não bloqueantes, por exemplo quando parâmetros estão fora dos intervalos normais |

    <Tip>
      Quando `cacheHit` é `true`, o resultado foi retornado do cache Redis e não recalculado. Resultados em cache são idênticos aos recém-calculados e contam para a cota de requisições do seu plano.
    </Tip>
  </Step>
  <Step title="Explore a referência completa da API">
    O endpoint de cálculo acima é um dos vários endpoints públicos disponíveis com a permissão `CALCULAR_FRETE`. A permissão `GERENCIAR_CARGAS` desbloqueia os endpoints de gestão de cargas para criação e acompanhamento de registros individuais de embarque.

    Consulte a [Referência da API](/api-reference/overview) para documentação completa de todos os endpoints, schemas de requisição, códigos de erro e modelos de resposta.
  </Step>
</Steps>
